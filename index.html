PCA in 2D

代码同时可以在我的github上找到，git@github.com:RobotLiu2015/machine-learning.git.如果需要交流可以发email到ypliurun@gmail.com.

import numpy as np
import matplotlib.pyplot as plt
import math

#数据要求：每列代表一个数据集，每行代表一个维度
def zeroMean(dataMat):
meanVal=np.mean(dataMat,axis=1) #按行求均值，即求各个特征的均值
meanVal = np.reshape(meanVal,(2, 1))
newData=dataMat-meanVal
return newData,meanVal

Step 0: Load data

dataMat = np.loadtxt(‘pcaData.txt’)#读取文件中的数据
plt.scatter(dataMat[0],dataMat[1])
plt.show()

Step 1a: Finding the PCA basis

newData,meanVal=zeroMean(dataMat)

print(meanVal)

covMat = np.cov(newData,rowvar=1)
eigVals,eigVects=np.linalg.eig(np.mat(covMat))#求特征值和特征向量,特征向量是按列放的，即一列代表一个特征向量
print(eigVals,eigVects)
plt.plot([0,eigVects[0,0]],[0,eigVects[1,0]],color=”blue”, linewidth=1.0, linestyle=”-“)
plt.plot([0,eigVects[0,1]],[0,eigVects[1,1]],color=”blue”, linewidth=1.0, linestyle=”-“)
plt.scatter(dataMat[0],dataMat[1])
plt.show()

Step 1b: Check xRot

eigValIndice=np.argsort(eigVals) #对特征值从小到大排序
eigValIndice = eigValIndice[::-1]
eigVects=eigVects[:,eigValIndice]#所有的特征向量重新按照大小排布
Xrot = np.dot(eigVects.T,newData)
plt.scatter(Xrot[0],Xrot[1])
plt.show()

Step 2: Dimension reduce and replot

U = eigVects + 0 #不能够直接赋值，因为直接赋值之后地址的值也会变化
U[:,1] = [0]
Xhat = np.dot(U,Xrot)
plt.scatter(Xhat[0],Xhat[1])
plt.show()

Step 3: PCA Whitening

epsilon = 0
eigVals = eigVals + epsilon
print(eigValIndice)
eigVals = eigVals.reshape(len(eigVals),1)
eigVals = eigVals[eigValIndice,:]#所有的特征值重新按照大小排布
Xpcawhite = Xrot/np.sqrt(eigVals)
plt.scatter(Xpcawhite[0],Xpcawhite[1])
plt.show()

Step 4: ZCA Whitening

Xzcawhite = np.dot(eigVects,Xpcawhite)
plt.scatter(Xzcawhite[0],Xzcawhite[1])
plt.show()

机器学习之——python开发环境

首先说明一下，博主目前在学习机器学习的算法，有使用python练习《机器学习实战》中的例子，同样是初学的小伙伴一起哦！

软件安装

虽然已经熟悉使用Linux系统，但是鉴于在windows下面搜索信息和使用各种软件比较方便，我就在windows系统下安装python开发环境，简单说明一下软件的安装。目前使用的系统是win7，python版本是3.4.3。

安装在win7下python

按照提示步骤安装python-3.4.3.msi，选择所有的都安装，这样包含pip，easy_install都会安装上，方便后面安装其他的软件包。

安装matplotlib

matplotlib安装需要安装一些依赖的软件，因为matplotlib在使用时候会调用其他软件的库。Matplotlib 安裝程式 :

  • matplotlib-1.4.3.win32-py3.4.exe 或matplotlib-1.4.3-cp34-none-win32.whl (用法：pip install matplotlib-1.4.3-cp34-none-win32.whl)

一起安裝以下套件：

  • python-dateutil-2.2.win32-py3.4.exe 或python_dateutil-2.4.2-py2.py3-none-any.whl (用法：pip install python_dateutil-2.4.2-py2.py3-none-any.whl)
  • pyparsing-2.0.2.win32-py3.4.exe或pyparsing-2.0.3-py3-none-any.whl(用法：pip install pyparsing-2.0.3-py3-none-any.whl)
  • six-1.6.1.win32-py3.4.exe 或six-1.9.0

检验安装效果方法：
http://liam0205.me/2014/09/11/matplotlib-tutorial-zh-cn/

安装PyCharm

使用python，尤其是初学者，最好使用IDE来开发，因为这样可以多次的调试。当然Ipython使用起来也是很不错的！我选择使用PyCharm，其分为两个版本，去到官网搜索安装就好。专业版包含的工具比较多，但是需要破解，不过不用担心，在我TC什么都能找到的！

安装其他软件包

一些软件包的安装使用pip就变得很简单，一个指令就可以完成。在windows下，安装python-3.4.3.msi软件之后，会相应的安装一系列的软件，具体的在安装的过程中可以看出来。其中pip就已经安装，但是在windows下使用pip的话要使用cmd，进入pip.exe所在的文件夹，然后pip安装whl文件。也可以直接在cmd里面安装包的名称。

如果pip因为错误原因被卸载，可以使用easy_install安装pip：easy_install pip。

安装其他软件：

  • scipy-0.15.1-win32-superpack-python3.4.exe
  • numpy-1.9.2-win32-superpack-python3.4.exe

一个说明安装关联软件方法的网站：http://www.math.ntnu.edu.tw/python/

所有的软件安装完之后，就可以开始编程了哦！

ARM-Linux开发框架（二）

首先对最近没有更新博客自我检讨一下，写博客是自我总结、提高和消化的过程，最近仅仅是在不停地去寻找食物，恰恰没有把食物消化好，罪过罪过！

开发框架更新

转回到正题！最近虽然课有点多，但是对于知识更加的渴望，也许因为是越得不到就越渴望吧！

之前写过一个开发框架，因为最近应用一个在windows下编译软件通过TCP/IP操作arm，这就使得之前使用arm连接linux主机的方式暴露出问题，老是插拔网线，搞得人神烦！终于还是在这件神烦事情刺激之下，寻找了一个新的方式（可见问题与困难才是进步的前提呀！）。

目前使用windows安装网卡，这样就多了一个网口，这样在windows下就可以通过网线对arm进行操作了。并且我们可以使用PuTTY通过telnet登录到arm的linux系统，完成基本的操作。但是这样暴露了一个问题，我宿主linux开发的程序无法方便的拷贝到arm。其实还是有解决方式的，但是限于我的电脑只有2G内存，就果断放弃使用虚拟机了，纠结呀，继续在寻找解决方法。

方法一

使用windows装linux虚拟机，虚拟机完成交叉编译，在windows下通过telnet对arm-linux进行操作。这样一台电脑就可以完成工作，windows可以联网查资料，方便小白开发。

方法二

那就是直接使用linux完成开发和调试工作。在linux之下也可以使用telnet连接到arm-linux，这样开发下载都可以轻松的解决。但是对于初学者来说，很多的知识还是通过windows搜索，包括办公软件，所以感觉小白还是不要选择这种方式。

ARM-Linux连接网络

下午吃饭之前有点时间，无聊的就尝试使用arm-linux连接互联网。目前情况是这个样子，arm-linux使用网线与windows连接。我们windows在网络管理里有桥接和分享的选择。我想分享对于使用win7自己分享wifi的童鞋来说不陌生。其实就是将分享的对象换为另一个本地连接就好，这样arm-linux就可以连接到互联网了，最方便的测试方式就是ping一下www.baidu.com了，如果连得通就是已经连接到互联网了。但是我在使用中出现了一个问题，那就是arm-linux能够连接互联网，但是我的windows无法访问网络了，一定是某个地方出现了问题，搞定之后再来分享！

以后要坚持写下自己的想法，养成良好的习惯，加油！RobotLiu！

我来评AmazingBox

坚决不写软文

在文章中可能会介绍到一些智能硬件，但是请大家相信RobotLiu绝对是带着欣赏、理智的眼光去写出自己对于这些智能硬件的理解的，绝非软文。

AmazingBox

第一次看到AmazingBox还是在X东的众筹的网站上，看到她第一眼就知道这是我一直苦苦追寻的那个（额，不要多想）！因为之前对于物联网多少会关注一点，发觉前期的物联网，大家都是在做智能开关等一些终端设备，自己家做自己的，没有一个统一的系统。也就是在看到了她之后，我感觉到希望来了，物联网总归会走向相同的接口，AmaingBox就做了这样一个工作。AmazingBox吸引我的就是她的框架，个人感觉物联网会走向统一的接口接入互联网，AmazingBox就在做着初探的工作，相信前途是光明的。
对与这款路由的基本的基本性能，鉴于个人基本知识的缺乏有些还不能了解深入。仅仅就AAmazing在室内的双网：wifi和Zigbee说明一下。Zigbee在室内的通信是没有问题的，就其带宽也是没有问题，因为Amazing应用Zigbee控制和进行数据交换的数据不是很大，是一些例如环境温度、空气参数等。这些数据是字节级的，这样Zigbee完全可以胜任。其他的例如网络摄像头等数据量大，就可以应用wifi无线或者是有线解决了。对于Amazing的这一点个人感觉还是很好的。
说几点个人感觉不够好的地方：目前看到Amazing推出的云服务，就现在市场行业认可度还不够的情况下，这个云的发展还是有些鸡肋的。还有Amazing开放的第三方开发的SDK，大家都趋之若鹜的去做社交软件了（夸张一点），在这样一个比较封闭的环境下，人才的投入还是不够呀，Amazing要自己投入足够的精力呀！还有很重要的一点，我想很多的做物联网的人来说也都在考虑这个问题，那就是跟目前的家电产品各下一代家电产品融合的问题，就目前控制一个灯可以，将来要做的事整套的智能家居系统，这样才会给消费者更好的生活体验。也希望未来有这样集合大家电控制产品的出现！
