linux下安装pycharm

在win7下使用pycharm编写UFLDL Tutorial的习题时候遇到了内存错误的问题，所以开始转战linux。就安装了ubuntu14.04，可是在python的一些开发包的安装上遇到了问题，经过了一段时间的煎熬总算是搞定了，学习新东西就是一个不断填坑的过程呀!

使用经历

linux下安装软件非常蛋疼的一个问题是软件的依赖问题，有些软件的安装依赖其他软件的安装，但是有时候需要安装的软件的依赖的版本跟系统里已经有的版本不一样，问题就变得严重了。这个问题主要应该是各个软件更新的速度是不一样的，所以在使用时候一定要选择好，要是有人带的话就好了，因为别人都已经把坑都给填过了，自己来就是一个酸爽的旅程。

我在使用时遇到的问题是这样子的，我安装的是14.04.3，它的一些最新的软件就导致一些依赖无法安装。我想要使用numpy，安装之后报错需要libapack，我再继续使用apt-get安装libapack报错依赖fortran3，继续安装fortran3那就暴露问题所在了，fortran3一定是依赖gcc-4.8-base也就是gcc-4.8-base (= 4.8.2-19ubuntu1)GCC, the GNU Compiler Collection (base package)。可是我的系统里面是4.8.4。看了网上很多安装numpy的帖子，都没有讲到这个问题，经过痛的教训之后我考虑到，这个问题是因为不同平台软件更新速度的问题。所以我考虑换一个第一版本的ubuntu，误打误撞看到了优麒麟，就本着安装试试的态度，结果很好，都安装上了（貌似在做广告呀！见谅哈）但是不知道这个号称根据国人习惯做的桌面系统使用起来会怎样，这样以后验证了。总结一下这里出现的问题主要是版本不匹配的问题，我使用的是14.04.2，gcc的版本正好，我猜想原装14.04.1和14.04.2应该也可以。

简单写一下我的安装过程，我使用的是python3.4，如果再需要其他的软件可以使用pip3安装。

sudo apt-get install python3-pip

sudo apt-get install python3-dev

sudo apt-get install python3-numpy

sudo apt-get install python3-scipy

sudo apt-get install python3-matplotlib
